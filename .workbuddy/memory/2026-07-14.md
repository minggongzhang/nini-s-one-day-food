## 修复：自定义标签输入 bug（字符被吞 / 仍提示请输入标签）

**根因**：controlled input 模式下，`value="{{ customTagValue }}"` + `bindinput` 每次输入都 `setData`，setData 异步导致竞态——快速输入时输入框被重置为旧值，字符被吞；`data.customTagValue` 与实际输入不一致，点击添加时读到空值。

**修复方案**：
- 移除 input 的 `value` 绑定（改为非受控输入）
- 用页面实例属性 `this._customTagValue` 存储输入值
- 添加标签后通过 `wx:if="{{ tagInputVisible }}"` 切换销毁重建输入框来清空内容
- 30ms 后重建，保持焦点连续

## 修复：单设备切换角色收不到通知

**用户反馈**：一台手机切换两个角色下单，收不到通知。

**根因分析**：
1. **缺少 config.json 权限声明**：三个云函数都没有 `config.json`，未声明 `subscribeMessage.send` 权限，`cloud.openapi.subscribeMessage.send()` 调用被拒

3. **同 openid 单设备测试限制**：`registerUser` 切换角色时更新同一条 user 记录，登录为女朋友时 DB 中没有 role=boyfriend 的用户，`notifyBoyfriends` 查不到人 → 新订单通知无法发送
4. 但"订单状态变更通知"（男朋友→女朋友）可以单设备测试：发送目标是 `order.userId`（下单时已记录），不依赖当前角色

**修复**：
- 为 createOrder、updateOrderStatus、sendSubscribeMessage 三个云函数添加 `config.json`（声明 subscribeMessage.send 权限）
- 三个云函数的 `miniprogramState` 从 'formal' 改为 'developer'

**单设备测试方案**：
- 方案A：两台手机/两个微信号（完整测试）

## 排查：-604101 错误（用户反馈测试通知按钮报此错）

**错误现象**：在微信开发者工具中点击「发送测试通知给我自己」后弹窗显示 `system error: error code: -604101`。

**根因（重要）**：
- `-604101 = function has no permission to call this API` 是云函数调用 openapi 权限不足
- **但最常见原因是微信开发者工具（DevTools）的云调用 Bug**：DevTools 调用 `cloud.openapi.subscribeMessage.send()` 时经常误报 -604101，**真机调试 100% 正常**
- 参考：微信开放社区多个开发者反馈"开发工具报错、真机正常"
- 真机也报 -604101 才是真的权限问题（config.json 未部署/未生效）

**修复**：
1. 前端 `pages/orders/index.js` 的 `onTestNotify` 增加友好错误处理：检测到 -604101 时弹出"这是开发者工具 Bug，请用真机测试"的提示，而不是显示冷冰冰的系统错误
2. 三个云函数（`createOrder`、`updateOrderStatus`、`sendSubscribeMessage`）的 `config.json` 之前已创建，权限声明 `subscribeMessage.send` 正确
3. `sendSubscribeMessage` 云函数错误处理增强：返回 `hint` 字段指引排查方向

**config.json 关键细节**（踩坑记录）：
- 写 `subscribeMessage.send`，**不是** `openapi.subscribeMessage.send`
- 必须"上传并部署：云端安装依赖"才能让 config.json 生效
- 权限配置有约 10 分钟缓存时间

**排查流程**：
- 先用真机（预览/真机调试）测试，能用就直接用
- 真机也报，才排查 config.json 和部署状态

## 修复：模板字段不匹配导致 data.thing14.value is empty

**用户反馈**：双设备测试男朋友未收到新订单通知，测试按钮报 `data.thing14.value is empty`（错误码 47003）。

**根因**：
- 我代码里假设的模板字段编号是 `thing01`/`thing02`/`amount01`/`character_string01`/`date01`/`phrase01`（占位假设）
- 用户实际模板字段编号为：
  - 新订单通知：`thing14`（下单人）、`thing13`（产品名称）、`amount4`（金额）、`character_string22`（订单号）、`time7`（订单时间）
  - 订单状态更新：`character_string1`（订单号）、`thing6`（商品名称）、`phrase3`（订单状态）、`amount4`（订单金额）、`thing5`（备注）
- 字段编号不一致导致微信返回 `47003` 参数不准确错误

**修复方案**：
1. 新增 `utils/templateHelper.js` 工具函数 `fillTemplateData`，根据模板字段定义自动补全缺失字段（默认填充 `无`/`¥0.00`/`无` 等）
2. 在 `utils/subscribeConfig.js` 中新增 `TEMPLATE_FIELDS` 配置，按实际模板字段列出完整字段列表（前端使用）
3. 在 `cloudfunctions/createOrder/index.js` 和 `cloudfunctions/updateOrderStatus/index.js` 中**内联**模板 ID、字段列表和 `fillTemplateData` 函数，避免云函数依赖项目根目录的 `utils` 文件
4. 两个云函数按实际字段编号构造模板数据，并调用本地 `fillTemplateData` 补全
5. 时间字段按微信 `time` 类型格式化为 `yyyy年mm月dd日 HH:MM:SS`

**注意**：
- 前端页面仍使用 `utils/subscribeConfig.js` 中的 `TEMPLATES` / `TEMPLATE_FIELDS`
- 云函数已内联配置，**以后换模板时需要同时改两个地方**：`utils/subscribeConfig.js`（前端） + `cloudfunctions/createOrder/index.js` 和 `cloudfunctions/updateOrderStatus/index.js` 顶部内联配置（云函数）

## 修复：云函数 require 外部 utils 导致下单/接单失败

**用户反馈**：真机测试时女朋友端下单显示"网络异常，请重试"，男朋友端点击接单显示"操作失败"。

**根因**：
- `createOrder` 和 `updateOrderStatus` 云函数通过 `require('../../utils/subscribeConfig')` 和 `require('../../utils/templateHelper')` 引用项目根目录的 `utils` 文件
- 微信云函数打包上传时只包含云函数目录内的文件，目录外的 `utils` 文件不会被打包到云端，导致云函数在云端运行时报"Cannot find module"错误，整个云函数执行失败
- 前端调用 `wx.cloud.callFunction` 失败，显示"网络异常，请重试"

**修复方案**：
1. 将模板配置和 `fillTemplateData` 工具函数内联到 `cloudfunctions/createOrder/index.js` 和 `cloudfunctions/updateOrderStatus/index.js` 中
2. 删除两个云函数中 `require('../../utils/...')` 的引用
3. 保留项目根目录 `utils/subscribeConfig.js` 和 `utils/templateHelper.js` 供前端使用
4. 重新部署 `createOrder` 和 `updateOrderStatus` 云函数

**后续模板变更**：
- 若后续模板 ID 或字段编号变化，需同时更新：
  - `utils/subscribeConfig.js`（前端页面）
  - `cloudfunctions/createOrder/index.js` 顶部内联配置
  - `cloudfunctions/updateOrderStatus/index.js` 顶部内联配置
