## 修复：菜品列表真机显示为单列的问题
- **文件**: `pages/food/index.scss`
- **原因**: `.food-list` 使用了 `gap: 24rpx` + `.food-item` 使用 `width: calc(50% - 12rpx)`。`gap` 在微信小程序真机 Flexbox 中兼容性差，加上 `calc(50% - 12rpx)` 的亚像素舍入问题，导致两个元素总宽度恰好或略超 100%，第二个元素被挤到下一行。
- **修复方案**: 移除 `gap`，改用 `justify-content: space-between` + `width: 48%` + `margin-bottom: 24rpx`，兼容性更好。
- **教训**: 微信小程序真机调试时，Flexbox 的 `gap` 属性不可靠，应优先使用 `justify-content: space-between` 或 margin 方案。

## Bug 修复：点击“加入购物车 / 去结算”提示“请先登录”
- **原因**: 客户端 Storage 里的 `userInfo` 字段是 `openid`（小写），但代码里判断的是 `userInfo.openId`（大写 I），大小写不一致导致登录状态判断失败。
- **修复**: 
  - 统一将客户端所有 `openId` 改为 `openid`（与云函数返回一致）
  - 在 `app.js` onLaunch 中增加旧数据迁移：若旧数据只有 `openId` 而无 `openid`，自动迁移并写回 Storage
  - 登录判断处兼容两种写法：`userInfo.openid || userInfo.openId`
- **涉及文件**: `app.js`, `pages/login/login.js`, `pages/food/index.js`, `pages/profile/index.js`, `pages/add-food/index.js`

## Bug 修复：自定义 TabBar 编译报错

- **现象**: `[ WXSS 文件编译错误 ] ./custom-tab-bar/index.wxss(13:17): unexpected \`{\` at pos 250`
- **原因**: `custom-tab-bar/index.wxss` 被写成了嵌套 SCSS 语法（如 `.custom-tab-bar { .tab-bar-list { ... } }`），但 `.wxss` 文件是小程序原生样式，不支持嵌套和 SCSS 变量导入；页面 `.scss` 才会被开发者工具预处理。
- **修复**: 将 `custom-tab-bar/index.wxss` 重写为平铺的纯 CSS/WXSS 语法，去掉 `@import '../styles/variables.scss'`，并把颜色值直接写死为 `#ff6b6b` / `#999999`。
- **涉及文件**: `custom-tab-bar/index.wxss`
- **教训**: 自定义 TabBar 组件必须使用 `.wxss` 文件并写原生 CSS，不能和页面一样使用 `.scss` 嵌套语法。
