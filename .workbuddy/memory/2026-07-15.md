## 菜品图标显示修复（最终方案）

**前因**：之前用 `binderror` + 图片/icon 叠加层方案，导致部分设备图片和 emoji 都显示异常（方块）。用户反馈不需要无图缺省图标。

**最终方案**：回到最简单的 `wx:if`/`wx:else` 互斥渲染，去掉 `binderror`、`imgErrors` 状态、叠加层逻辑：
- `imageUrl` 有值 → `<image>` 显示图片
- `imageUrl` 为空 → `<text>` 显示 emoji（`item.icon || '🍽️'`）

**改动文件**：`pages/food/index.wxml`（简化 food-image）、`pages/food/index.js`（删除 imgErrors data 和 onFoodImgError 方法）、`pages/food/index.scss`（恢复简单 flex 居中布局）。

## 修复：全局 emoji 图标无法显示 + orders 页面内容被注释掉

**根因1**：`app.scss` 中 `page` 的 `font-family` 只声明了西文字体（-apple-system, BlinkMacSystemFont 等），没有 emoji 字体。iPhone 16 上 WeChat 渲染引擎不会自动 fallback 到 emoji 字体，导致所有 emoji（🍔🍟🥤💕👤等）显示为方块/空白。

**修复1**：font-family 添加 `'Apple Color Emoji', 'Noto Color Emoji', 'Segoe UI Emoji'`。

**根因2**：`pages/orders/index.wxml` 第28行 `<!--<view class="test-notify-btn"...` 开了 HTML 注释但没闭合 `-->`，导致后续所有内容（状态筛选、订单列表、空状态、加载状态）全被注释掉。

**修复2**：正确闭合注释 `<!-- ... -->`。

**附带修复**：`pages/food/index.scss` 中 `.food-icon` 的 `line-height: 1` 改为 `1.2`，避免部分设备上 emoji 被裁切。
